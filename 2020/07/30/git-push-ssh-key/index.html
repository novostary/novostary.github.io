git-push-ssh-key
发表于2020-07-30|更新于2020-07-30|Tips
|阅读量:

配置方法

参考 How to specify different SSH keys for git push for a given domain

新建 ~/.ssh/config
# github-as-sth 可以随便取
Host github-as-sth
  HostName github.com
  User git
  IdentityFile ~/.ssh/your-ssh-key
  IdentitiesOnly yes

$ git remote add origin git@github-as-sth:novostary/hexo_github.git
$ git push -u origin master

一些错误及解决方案

git@github.com: Permission denied (publickey).

密钥不是 id_rsa，需要指定 SSH 密钥，也即本文的方法。

$ git push -u origin master
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

fatal: remote origin already exists

由于方法中用了一个假域名，但是已经指定了真实的 git 路径，所以，需要修改这个配置。

$ git remote add origin git@github-as-mm:novostary/hexo_github.git
fatal: remote origin already exists.
$ git remote -v
origin  git@github.com:novostary/hexo_github.git (fetch)
origin  git@github.com:novostary/hexo_github.git (push)

修改方法如下：

$ git remote set-url origin git@github-as-mm:novostary/hexo_github.git
$ git remote -v
origin  git@github-as-mm:novostary/hexo_github.git (fetch)
origin  git@github-as-mm:novostary/hexo_github.git (push)
$ git push -u origin master
